# Story 13: Toxicity Reports

Feature area: Analytics and Reporting

Story:
As a moderator, I want periodic toxicity reports, so that I can see trends over time.

Acceptance criteria:
- Reports summarize weekly toxicity metrics.
- Reports are generated on a schedule or on demand.

Feasibility rating: Low-Medium

Justification:
Requires persistent storage and scheduling.

Implementation notes:
- Start with on-demand reports that query stored metrics.

Modified story:
As a moderator, I want an on-demand toxicity snapshot, so that I can see current trend metrics without scheduled jobs.
